# Code of Conduct

## Our Pledge

We expect all participants in this project to treat one another with respect, professionalism, and courtesy. Everyone should be able to collaborate without harassment, discrimination, or intimidation.

## Expected Behavior

- Use respectful and constructive language.
- Focus feedback on code, design, and ideas, not on people.
- Be patient with questions, reviews, and different experience levels.
- Respect the project scope, decisions, and maintainers' guidance.

## Unacceptable Behavior

- Harassment, threats, insults, or personal attacks.
- Discriminatory language or behavior based on gender, race, ethnicity, nationality, religion, disability, age, appearance, or other personal characteristics.
- Publishing private information without consent.
- Deliberate disruption of discussions, issues, reviews, or pull requests.

## Project Standards

- Keep discussions focused on the security system and its documentation or code.
- Be clear and factual when reporting bugs or reviewing changes.
- Assume good faith, but do not ignore harmful behavior.

## Enforcement

Project maintainers may remove, edit, or reject contributions that violate this code of conduct. Repeated or severe violations may result in temporary or permanent removal from project participation.

## Reporting Issues

If you experience or witness behavior that violates this code of conduct, report it through the project's maintainers or the repository issue tracker.

## Scope

This code of conduct applies to repository discussions, pull requests, issues, code reviews, and any other project-related communication.
